# 🚀 مشروع زودها SMM - تحديث قاعدة البيانات والبنية التحتية بالكامل
# ZWDHA SMM - Complete Database Modernization & Infrastructure Build

تم تطوير هذا المشروع وتحديثه بالكامل ليصبح نظامًا متكاملًا جاهزًا للعمل والإنتاج في البيئات السحابية (Production-Ready)، مع الحفاظ التام والكامل على جميع ميزات واجهة المستخدم وتصميمها الأصلي وتجربة المستخدم السابقة دون المساس بأي عنصر مرئي.

---

## 🛠️ التحديثات والتحسينات الرئيسية (Modernization Core Features)

### 1. قاعدة البيانات وإدارة الجداول (Database & Prisma ORM)
* **استبدال SQLite بـ PostgreSQL:** تم الانتقال بالكامل لقاعدة بيانات علائقية قوية ومستقرة، وتم ربطها بـ **Cloud SQL** في بيئة Google Cloud بنجاح.
* **الاعتماد على Prisma ORM:** توفير كتابة استعلامات عالية السرعة والأداء وتجنب ثغرات حقن الاستعلامات (SQL Injection).
* **بناء العلاقات بين الجداول (Database Relations):**
  * تم ربط جدول الطلبات (`Order`) بجدول الباقات (`Package`) لضمان تكامل البيانات.
  * تم ربط جدول الطلبات بجدول العملاء (`Customer`) تلقائيًا عند إنشاء أي طلب جديد بناءً على رقم الهاتف.
  * إنشاء جدول سجل حالات الطلب (`OrderHistory`) لتتبع التغيرات التاريخية لكل طلب بدقة.
  * إنشاء جدول سجلات الأمان والنشاطات الإدارية (`ActivityLog`).

### 2. نظام إدارة العملاء والطلبات المتقدم (Advanced CRM & Orders)
* **ربط تلقائي للعملاء:** يتم فحص رقم هاتف العميل (`whatsappNumber`) تلقائيًا عند كتابة الطلب؛ إذا كان العميل جديدًا يتم إنشاء سجل خاص به، وإذا كان حاليًا يتم تحديث بياناته وحفظ الطلب في سجله التراكمي.
* **إحصائيات كاملة للعميل:** تسجيل (عدد الطلبات، إجمالي المبالغ المنفقة، تاريخ آخر طلب، وسجل الطلبات التاريخي الكامل له).
* **سجل حالات الطلب التاريخي (Order History Logs):** يسجل النظام تلقائيًا تاريخ ووقت كل عملية تعديل لحالة الطلب أو حالة الدفع (Pending, Paid, Rejected) والمسؤول الذي قام بالتعديل مع كتابة ملاحظات مخصصة.

### 3. الكوبونات المتقدمة ونظام الإيميلات (Advanced Coupons & Mailer)
* **تحديث نظام الكوبونات:** يدعم الكوبون الآن ميزات متقدمة تشمل:
  * نوع الخصم (نسبة مئوية `PERCENT` أو مبلغ ثابت `FIXED`).
  * حد أدنى لقيمة المشتريات لتفعيل الكود (`minPurchase`).
  * تاريخ انتهاء صلاحية الكوبون (`expiresAt`).
  * الحد الأقصى لعدد مرات استخدام الكوبون الإجمالي (`maxUses`) ومتابعة العداد الفعلي لاستخدامه (`useCount`).
* **لوحة SMTP متكاملة:** إمكانية التحكم الكامل وتفعيل أو إيقاف رسائل الإيميل للمشتري والإدارة، ودعم فحص الإرسال مباشرة من لوحة التحكم بضغطة زر عبر زر **Send Test Email** المدمج في لوحة التحكم.

### 4. حماية أمنية قوية (Advanced Security Controls)
* **حماية Helmet:** تفعيل حماية ترويسات HTTP لتجنب هجمات XSS وحقن المحتوى.
* **تفعيل CORS:** تحديد نطاقات الاتصال المقبولة وتأمين نهايات الـ API.
* **سجل النشاطات الإدارية (Activity Security Logs):** يسجل النظام كل تفاعل حساس للإدارة، مثل تعديل الحالات، تفعيل الكوبونات، تغيير كلمات المرور، أو إرسال إيميل تجريبي، مع توثيق (نوع الإجراء، اسم المسؤول، الـ IP الفعلي للجهاز، وتوقيت العملية بالملي ثانية).

## 📂 هيكل المجلدات والملفات المضافة (Infrastructure Files)

1. **`prisma/schema.prisma`**: النموذج المحسن لقاعدة بيانات PostgreSQL الذي يضم جميع الجداول والعلاقات الجديدة.
2. **`Dockerfile`**: ملف بناء الحاوية الذكي متعدد المراحل (Multi-stage Build) لتقليل حجم الحاوية ورفع الأداء والسرعة في التشغيل والإنتاج.
3. **`docker-compose.yml`**: ملف الأوركسترا لإنشاء الحاوية الخاصة بالتطبيق وحاوية قاعدة بيانات PostgreSQL 15 مستقلة وربطهما تلقائيًا بضغطة زر.
4. **`.env.example`**: ملف نموذج المتغيرات البيئية موضح به إعدادات الربط بقاعدة البيانات والـ JWT وغيرها.
5. **`run-db-push.js`**: سكربت مساعد لتحديث هيكل الجداول مباشرة على خادم قاعدة البيانات السحابي دون الحاجة لـ Migrations معقدة أثناء التطوير الأولي.

## 🚀 تشغيل المشروع محلياً عبر Docker (Local Development via Docker)

يمكنك الآن تشغيل المشروع وقاعدة البيانات محلياً بشكل فوري وبأمر واحد فقط:

```bash
# بناء وتشغيل الحاويات في الخلفية
docker-compose up -d --build
```

سيقوم Docker بتنزيل خادم PostgreSQL وبناء التطبيق وتشغيلهما معًا على المنفذ `3000`.

## 💾 أوامر التحكم بقاعدة البيانات وتحديثها (Database Synchronization Tools)

عند إجراء أي تعديل مستقبلي على ملف `prisma/schema.prisma`، يمكنك مزامنة قاعدة البيانات عبر الأوامر التالية:

### لبيئة التطوير المحلية (Local Dev Sync):
```bash
npx prisma db push
```

### لإنشاء هجرة وتوثيق التغييرات (Generate Migrations for Production):
```bash
npx prisma migrate dev --name init_new_schema
```

## ☁️ خطوات النشر في السيرفر السحابي (Cloud Run Deployment)

يتم النشر مباشرة في Google Cloud Run باستخدام الحاوية المصممة في `Dockerfile`:

1. قم بتجهيز قاعدة بيانات PostgreSQL على خادم سحابي (مثل Cloud SQL).
2. قم برفع الحاوية وبنائها على خادم الحاويات السحابي:
   ```bash
   gcloud builds submit --tag gcr.io/[PROJECT_ID]/zwdha-app
   ```
3. انشر التطبيق على Cloud Run وقم بتمرير المتغير البيئي `DATABASE_URL` وخادم قاعدة البيانات السحابي:
   ```bash
   gcloud run deploy zwdha-app \
     --image gcr.io/[PROJECT_ID]/zwdha-app \
     --platform managed \
     --region europe-west3 \
     --set-env-vars DATABASE_URL="postgresql://[USER]:[PASSWORD]@[DB_HOST]:5432/[DB_NAME]"
   ```
